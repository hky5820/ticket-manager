# 🎫 티켓 보관함

내가 예매한 공연 티켓을 정리해서 안 까먹기 위한 모바일 친화 웹앱.
설치·서버·회원가입 없이 **`index.html` 파일 하나**로 동작합니다.

## 기능
- 예매처 선택 (티켓링크·멜론·세종·YES24·클립서비스·NOL·샤롯데 기본 제공, **+ 버튼으로 직접 추가**)
- 공연명 / 공연 날짜 / 매수 / 가격 / 자리(등급·층·구역·번호) / 메모
- 자리 여러 개 추가·삭제, 티켓 자유롭게 추가·수정·삭제
- 상단 통계(공연 수·총 매수·총 지출), 공연명/예매처 검색
- 데이터는 브라우저에 자동 저장(localStorage)
- **백업 내보내기/불러오기**(JSON)로 기기 이동·데이터 보관

## 바로 쓰기
`index.html`을 더블클릭해서 브라우저로 열면 끝. 휴대폰에서는 아래처럼 배포해서 쓰면 편해요.

## 무료 배포 (셋 중 하나)

### 1) Netlify Drop — 가장 쉬움 (계정도 선택)
1. https://app.netlify.com/drop 접속
2. 이 `TicketManager` 폴더를 통째로 드래그&드롭
3. 나오는 주소(예: `https://랜덤이름.netlify.app`)를 휴대폰에서 열고 홈 화면에 추가

### 2) GitHub Pages
1. GitHub에 새 저장소 만들고 `index.html` 업로드
2. Settings → Pages → Branch를 `main` / `/root`로 지정 후 저장
3. 몇 분 뒤 `https://아이디.github.io/저장소이름/` 로 접속

### 3) Vercel
1. https://vercel.com 로그인 → Add New → Project
2. 이 폴더를 올리면 자동 배포, 주소 발급

## 휴대폰 홈 화면에 앱처럼 추가
- **아이폰(사파리):** 공유 버튼 → "홈 화면에 추가"
- **안드로이드(크롬):** 우측 상단 ⋮ → "홈 화면에 추가"

## ☁️ 여러 기기 자동 동기화 (이미 연결됨)

백엔드(무료 Supabase)는 **이미 앱에 연결해 두었습니다.** 테이블·보안정책 생성, Project URL·키
입력까지 끝난 상태라, 남은 건 **회원가입/로그인 한 번**뿐이에요.

1. 앱을 열고 우측 상단 **`로컬 저장` 버튼** 클릭
2. **회원가입** 탭에서 이메일 + 비밀번호(6자 이상) 입력 → **회원가입**
3. 버튼이 **`동기화 켜짐`** 으로 바뀌면 완료 ☁️

> 로그인하면 이 기기에 있던 로컬 티켓을 클라우드로 올릴지 물어봐요. 이후 **다른 기기(폰 등)에서도
> 같은 이메일·비밀번호로 로그인**하면 목록이 그대로 나타납니다.

**참고**
- 연결된 프로젝트: Supabase `ticket-manager` (서울 리전). 코드에 들어있는 키는 브라우저 공개용
  **publishable** 키라서 앱에 넣어도 안전합니다(각 계정은 **본인 티켓만** 볼 수 있게 Row Level
  Security 적용됨). `secret` 키는 앱에 넣지 않았습니다.
- 가입 시 확인 메일 인증이 뜬다면, Supabase 대시보드 **Authentication → Email → Confirm email**을
  꺼두면 즉시 로그인됩니다(개인용 권장).

## 데이터 관리 팁
- 동기화를 설정하지 않으면 데이터는 **그 브라우저에만** 저장됩니다. 이때는 한쪽에서
  **백업 내보내기** → 다른 쪽에서 **백업 불러오기**로 옮기세요.
- 브라우저 캐시/데이터를 지우면 로컬 기록이 사라지니, 동기화 전이라면 가끔 백업을 내보내 두세요.
